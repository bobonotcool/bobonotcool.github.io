第二步，编辑并发布自己的博客
首先介绍一下，我所了解的在hexo发布博客的方式，有如下几种：
1、原始方式：
在我们搭建博客时创建的博客文件夹下面，source/_posts/目录下存放着所有博客文章的Markdown文件，初始化只有一个hello-world.md文件。
（1）如果要编辑博客，首先，（1.1），在根目录下,打开git bash，如下图中空白处右击鼠标选择Git bash here，
（1.2），在Git中输入命令：hexo new MyArticle，成功后我们会在source文件夹下的_posts文件夹中找到我们刚刚创建的文件MyArticle.md，或者就是直接手动在_posts文件夹下创建*.md文件的博客，但是我们自己要把头信息给补上，否则启动服务器之后我们的这篇文章会没有标题和时间等头信息，大家可以自己试试效果。
（1.4），用记事本或者各种编辑器打开MyArticle.md我们会看到一些hexo自动帮我们生成的头信息，我们可以在下面的行随便写一些东西，然后保存，
（1.5），在命令行，利用hexo g来生成，hexo d部署；
（2）如果要删除博客，则直接把_posts目录下相应的.md文件删除，再执行上述命令即可。
（3）如果要在博文中插入图片，可以把图片统一放置在source/images目录下，然后在使用时用下述方式引用：
2、使用hexo-admin插件管理：
（1）安装插件，在命令行输入：npm install –save hexo-admin
（2）启动服务器，在命令行输入：hexo server -d
（3）打开localhost:4000/admin/
开始编辑博文。
最后别忘了重新提交和部署。
PS：关于hexo-admin插入图片：
hexo-admin可以直接复制图片粘贴，然后自动下载到source/images目录并重命名。但在Windows中粘贴后会出现裂图。这时就需要手动把括号中的前后两个斜杠去掉，就能正常显示。
